# 🚀 DevOps / SRE Notes

## Linux Boot Process (Very Important)

Every Linux server follows this sequence:

```
Power On
      ↓
BIOS / UEFI
      ↓
Boot Loader (GRUB)
      ↓
Linux Kernel (vmlinuz)
      ↓
initrd / initramfs
      ↓
systemd (PID 1)
      ↓
Services Start
      ↓
User Login
```

As a DevOps or SRE engineer, understanding this flow helps troubleshoot systems that fail to boot.

---

## Why `systemd` Matters

Most modern Linux distributions use **systemd** to:

- Start services
- Stop services
- Restart services
- Enable services on boot
- View logs
- Manage system targets

Common commands:

```bash
systemctl status nginx
systemctl restart nginx
systemctl stop nginx
systemctl enable nginx
systemctl disable nginx
```

---

## Why Logs Matter

Logs are one of the first places engineers check during incidents.

Examples:

- Application crashes
- Server boot failures
- Authentication issues
- Disk errors
- Service failures

Important log locations:

```text
/var/log/messages
/var/log/syslog
/var/log/auth.log
```

---

## Kernel Logs

Useful command:

```bash
dmesg
```

Commonly used to diagnose:

- Boot failures
- Driver problems
- Hardware issues
- USB detection
- Disk errors
- Memory problems

---

## Log Rotation

Without log rotation:

- Log files grow indefinitely.
- Disks can become full.
- Applications may stop working.

`logrotate` automatically:

- Compresses old logs.
- Deletes outdated logs.
- Creates new log files.

This is critical in production systems.
